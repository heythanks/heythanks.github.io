关于HTTP

发表于 2018-12-25

初识HTTP协议

HTTP,英文全称是HyperText Transfer Protocol,中文释义就是超文本传输协议。设计HTTP最初的目的是为了提供一种发布和接受HTML页面的方法。作为一个靠谱的前端，了解HTTP是义不容辞的。

相识HTTP协议

何为相识HTTP协议呢？比如你跟某某相识，能用到相识这个词，不说相互之间有多知根知底，至少你是知道这个人，大概的为人如何？有什么特点？对，这里就是要介绍HTTP的特点，如下：

1、HTTP协议用于客户端和服务器之间的通信

HTTP协议用于客户端与服务器之间通信，在一条通信线路上必定存在一端客户端，另一端口服务器端。有时候两台计算机客户端和服务器端的身份可能互换，用HTTP协议能够明确区分身份。

2、通过请求和响应的交换达成通信

两端通过请求和响应的交换达成通信。举个小栗子：小明去商店买酱油，这次交易的过程肯定是小明跟老板说： “我要一瓶酱油”。老板听到了小明说得话然后从商店找出一瓶酱油给小明，小明拿着酱油开心的离开了…
这个过程中，小明就是通信过程中的客户端角色，他要买酱油的这个过程就是请求，老板作为服务器端角色，他接收到请求了，然后理解请求，再作出响应。

3、HTTP是不保存状态的协议

首先给你们讲个笑话

小鱼问大鱼：“妈妈，为什么他们说鱼的记忆只有七秒啊？”大鱼说：“你刚才说啥？”小鱼说：“啥？”大鱼说：“干啥啊？”小鱼说：“咋地了？”

不知道大家能不能从这个栗子中理解，什么叫不保存状态也就是无状态。虽然这里有个小bug,既然只有七秒记忆怎么知道这是他妈呢？这个，我也不知道，有兴趣的可以咨询生物学家或者忽略：）。我这里想表达的是，每一次通信过程中，都是一个全新的请求与响应，并不知道上一次发生了什么。这个性质对于我们的业务处理是不太方便的，当然这样设置有这样设置的原因。比如，用户登录一家购物网站，即使他跳到该站其他的页面后，也要继续保持登录。虽然HTTP是无状态的协议，但为了实现保持状态的功能，于是引入了Cookie技术。

4、请求URI定位资源

URI（Uniform Resource Identifer）统一资源标识符，这种标识允许用户对网络中的资源通过特定的协议进行交互操作。URI最常见的形式是统一资源定位符（URL）也就是我们平时上网时浏览器中地址栏的内容，俗称链接。另外一种就形式就是统一资源名称（URN）,至于具体解释大家可以上维基百科查询。

用浏览器中开发者工具解剖HTTP请求

  1. 打开Chrome浏览器中的开发者工具
  2. 选中面板上的Network选项
  3. 输入网址打开网站
  4. 点击选中Network中的第一个请求（左侧弹出一个面板）
  5. 查看左侧面板中的条目有General,Response Headers,Request Headers,Query String Parameters。
    要看请求直接展开Request Headers,点击view source,跟Request Headers同一行，展开才有，如果旁边是view parsed那就不用点击了，里面的内容便是http请求的组成。同理Response Headers是http响应的组成。
    注：Request Headers与Response Headers中的第一行是part 1,其他的key: value形式的都是part 2,part 3换行所以没显示。请求的第四部分在Query String Parameters中，响应的第四部分可以通过左侧面板上侧tab切换到Preview或者Response查看。
